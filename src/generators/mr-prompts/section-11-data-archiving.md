# Section 11 — Data and Metadata Archiving: Specific Instructions

You are generating **Section 11 (Data and Metadata Archiving)** of an Agricultural Census Metadata Review.

## Keywords for evidence retrieval

archive, database, storage, repository, metadata, microdata, cartography, digital, access, NADA, DDI, format, dissemination platform, website, open data

## What to cover

Describe how census data and metadata are stored and made available after the census. Address all of the following where documented:

1. **Final database format** — state the format in which the final micro-dataset is stored (e.g. CSV, SPSS .sav, Stata .dta, SAS, relational database). If multiple formats are available, list them.

2. **Availability of digital cartography** — state whether digital maps (e.g. enumeration area boundaries, sampling frame maps) are maintained and available, and in what format (e.g. shapefile, GeoJSON).

3. **Data access platforms and repositories** — describe the platforms or repositories through which census data and metadata may be accessed (e.g. national statistical office website, NADA catalogue, FAO repository, open data portal). Include URLs where documented.

4. **Metadata documentation** — note whether formal metadata documentation exists (e.g. a DDI record, a data dictionary, a technical report), and whether it is publicly available.

## Writing rules for this section

- Include URLs only when explicitly stated in the source documents.
- Do not describe dissemination publications (reports, tables) — those belong in Section 13.
- Do not describe quality assurance mechanisms — those belong in Section 10.
- If archiving arrangements are not documented, write exactly: *"Information on this point was not available in the source documents provided."*
- Write in formal, neutral, third-person prose.
