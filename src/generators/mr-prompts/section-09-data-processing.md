# Section 9 — Data Processing: Specific Instructions

You are generating **Section 9 (Data Processing)** of an Agricultural Census Metadata Review.

## Keywords for evidence retrieval

processing, validation, editing, imputation, correction, error, cleaning, consistency, coding, entry, tabulation, database, staff, supervisor, software, range check, logic check, outlier

## What to cover

Describe the post-collection data processing steps in the order they were applied. Address all of the following where documented:

1. **Built-in validation rules** — describe any automatic validation rules applied at the point of data entry (e.g. range checks, mandatory field controls, consistency rules embedded in the CAPI application).

2. **Post-collection consistency checks** — describe checks performed after data were submitted to the central system (e.g. cross-variable logic checks, outlier detection, comparison against frame data).

3. **Procedures for identifying and correcting errors** — describe how errors identified during editing were resolved (e.g. re-contact with enumerator or respondent, imputation, supervisory review, deletion).

4. **Roles of staff involved in data correction** — identify which categories of staff (e.g. data editors, supervisors, subject-matter specialists) were responsible for reviewing and correcting data.

5. **Data management systems** — name the software or systems used for data storage, editing, tabulation, and dissemination (e.g. CSPro, SPSS, SAS, a custom system).

## Writing rules for this section

- Describe processing steps in chronological order where possible.
- Use exact software names when documented.
- Do not describe data collection technology — that belongs in Section 8.
- Do not describe quality assurance mechanisms (training, supervision, back-checks) — those belong in Section 10.
- If processing steps are not documented, write exactly: *"Information on this point was not available in the source documents provided."*
- Write in formal, neutral, third-person prose.
